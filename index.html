Export Information

On Typst and LaTeX exportation

LaTeX (TikZ) and Typst (CeTZ) natively support wavy and spring-like paths for photons and gluons. However, when drawing highly curved or high-multiplicity propagators, native decorations can produce graphical artifacts due to phase shifts between the inner and outer strokes.

To prevent this, FeynDrawGram automatically falls back to an explicit, point-by-point path generation for the entire diagram when such complex lines are detected. While this ensures perfect pixel-to-pixel consistency with the web preview, it produces larger code that takes longer to compile and is harder to edit manually. If you prefer, you can force this high-fidelity export for all diagrams by checking the "Always use explicit waves" option above.

LaTeX dependencies

To compile the exported .tex file, you must include into the preable: 
\usepackage{tikz}
\usepackage{xcolor}
\usetikzlibrary{decorations.pathmorphing}
\usetikzlibrary{decorations.markings}
\usetikzlibrary{patterns}
\usetikzlibrary{patterns.meta}
\usetikzlibrary{arrows.meta}
After that, you can import your Feynman diagram with \input{feynman_diagram.tex}.
